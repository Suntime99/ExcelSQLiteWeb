# ExcelSQLite 优化与整合实施清单备查 (Optimization Plan)

## 1. 遗留优化项与 C# 后端依赖功能

以下是在前端界面调整完毕后，涉及深层逻辑重构及需要 C# 后端配合的优化项：

### 1.1 源文件解析与预览增强（“入湖前”体验）
- **数据嗅探与预览**：
  - 前端：已在【项目管理】的对象管理表格中新增【预览数据】按钮。
  - C# 后端需求：需实现 `action: 'previewData'` 接口，根据传入的 `filePath` 和 `sheet` 读取前 50 行数据，并将 JSON 结构（或 HTML table）回传给前端，前端负责弹窗渲染。
- **表头推断冲突预警**：
  - C# 后端需求：在解析工作表结构（即获取工作表列表时，或新增接口）提前检测表头，如果存在“空表头”或“重名表头”，向前端发出 `warning`。前端接收到警告后在 UI 上进行醒目的标红提示。

### 1.2 导入任务的队列管理与容错（“入湖中”体验）
- **更精细的进度反馈**：
  - C# 后端需求：目前批处理每万行提交一次进度，若能将 `current_row / total_rows` 的确切数值实时抛给前端，前端可将进度条改为真实百分比进度，避免超大文件“卡死”假象。
- **断点续传与失败跳过 (Import Log)**：
  - C# 后端需求：针对“多表并发或合并导入”，若其中某表出错，建议仅回滚报错表（或错误追加行），并继续执行队列中的下一张表。执行完毕后，回传一个包含成功/失败条目、失败原因的 JSON 数组。
  - 前端：新增“导入报告弹窗”，在执行向导结束后，展示清晰的“成功 x 张表，失败 y 张表（附带报错原因，如类型转换错误）”的明细表格。
- **异步后台任务**：
  - 架构调整：这是一个较重的基础设施级重构，需将 C# 导入任务与前端 UI 完全解耦，支持将导入任务最小化为右下角的“后台任务抽屉”。

### 1.3 多表合并追加的类型一致性（Schema Drift）
- **动态对齐 (自适应合并)**：
  - C# 后端需求：当用户选择【合并表数据导入】并将多个 Excel 表强行合并到同一个物理表时，如果第 N 个追加表出现了基础表中不存在的新列 C，SQLite 层应拦截 `INSERT` 异常，并在捕获后自动执行 `ALTER TABLE 基础表 ADD COLUMN C`；如果是基础表有的列而追加表没有，底层 SQL 引擎需以 `NULL` 补齐。
- **追加前的主键校验**：
  - C# 后端需求：Upsert 和 Dedupe 策略强依赖 SQLite 主键或唯一索引。后续考虑在新建表（首表）时，允许用户在前端勾选指定字段为 Primary Key，并生成相应的建表 DDL。

### 1.4 状态同步极致化 (Event-Driven Refresh)
- **静默刷新机制**：
  - 现状：前端已经支持了大部分操作后刷新 `renderSourceFiles()`。
  - C# 后端需求：针对任何导致 SQLite 物理表结构或数据发生变动（新增、删除、改名、重载）的操作，C# 执行完毕后，向前端主动推送一条 `{ action: 'db_changed' }` 的广播消息。
  - 前端：收到广播后，静默触发 `renderSourceFiles()`、下拉框更新等，取代目前的强制弹窗确认或手动刷新按钮。
