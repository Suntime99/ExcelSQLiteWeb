## v101 验收清单（最短闭环）

> 目标：验证“所见即所得 + 链路不丢 + 一键联动”是否真正跑通。

### 0. 前置
- 确保已保存/加载一个项目（【项目管理】页：`保存为项目` 或 `加载项目`）。
- 顶栏应显示：📌 当前项目、📄 当前主表、🧭 输出模式。

---

## A. 文件/文件夹 → SQLite 导入（含批量策略）

### A1. 元数据扫描（文件夹）
1. 打开【元数据扫描】选择“指定文件夹”，选择一个包含多个 Excel 的目录
2. 点击“开始扫描”
3. 在“扫描结果”卡片里设置：
   - 批量工作表选择：`每文件仅第一张 / 每文件全部工作表 / 按关键字匹配`
   - 主表选择策略：`第一文件第一张 / 按文件名关键字 / 按工作表名关键字`
4. 点击“加入并导入SQLite”

**验收点：**
- 源文件清单自动批量生成（重复文件自动跳过）
- 主表被自动选中（符合策略），顶栏📄 主表更新
- SQLite 导入完成后：可在各模块（文件分析/工作表分析/关系识别）看到表/字段

### A2. 元数据扫描（当前SQLite）
1. 【元数据扫描】选择“当前SQLite”
2. 点击“开始扫描”

**验收点：**
- 按钮联动正确：文件类按钮隐藏（加入源文件/导入SQLite）
- 结果表/视图列表可见，导出 HTML 可用

---

## B. 文件分析（SQLite 口径）→ 一键关系识别

1. 进入【文件分析】，切换到 `SQLite（内存库）` 口径
2. 点击 `▶ 解析`
3. 点击 `一键关系识别`

**验收点：**
- 自动跳转到【关联关系识别】页
- 表多选已自动勾选（来自文件分析结果）

---

## C. 关联关系识别 → 生成草稿（含组合键）

1. 【关联关系识别】配置采样参数（或用顶栏“采样设置”统一配置）
2. 点击“开始识别”
3. 选择：
   - 单条：点某行“生成草稿”
   - 批量：勾选多行 → “批量推送到多表关联/生成 MTJ 草稿”

**验收点：**
- 识别完成后，切换页面再回来/重启项目：能自动恢复上次结果
- 批量推送时：
  - 单字段关系：ON 自动按字段对接
  - 组合键/复杂关系：ON 以“表达式模式”生成，并提示你在 Step2 人工校验

---

## D. 多表关联（MTQ）→ 锁定 SQL → 自动保存

1. 在【多表关联】Step2 检查 ON（尤其是表达式 ON）
2. 继续到锁定 SQL 操作，点击“锁定 SQL”

**验收点：**
- 若开启“项目自动保存”：锁定后 2 秒内自动保存到项目 `.mtj.json`（不需手点保存）
- 若关闭“项目自动保存”：不会自动写入项目文件，仅保留当前会话配置

---

## E. 多表查询/多表统计：受控模式边界

1. 在【多表查询】或【多表统计】中勾选“受控模式”
2. 输入包含 `WITH ...`（CTE）或子查询的 SELECT SQL
3. 执行

**验收点：**
- CTE 不会被误判为“未锁定表”
- 若确实不通过白名单校验，会弹出“是否一键关闭受控模式并继续执行”

---

## F. 导出一致性

检查以下导出按钮文案/行为一致：
- 文件分析：导出分析报告（HTML）
- 工作表分析：导出分析报告（HTML）
- 元数据扫描：导出分析报告（HTML）
- 关系识别：报告导出/枚举导出（Excel 打开）

---

## G. SQL 查询分析（SQL 实验室）冒烟清单

### G1. 基础执行 + 远程分页
1. 进入【SQL查询分析】，默认应在【编辑】视图
2. 执行：`SELECT 1 AS a, 'x' AS b;`
3. 点击【结果】查看结果
4. 设置分页大小 100/200，点“下一页/上一页”（若当前结果不足可临时用大表 SQL）

**验收点：**
- 执行成功，结果列/行正确
- 远程分页下：筛选/排序会触发重新拉取（不会卡死）

### G2. 表格交互
1. 在结果表里拖拽列头右侧分隔条调整列宽（刷新/重进页面应记忆）
2. 拖拽列头进行列顺序重排（刷新/重进页面应记忆）
2. 点击“列”隐藏/显示列（导出应遵循隐藏列）
3. 点击“冻结”冻结前 0/1/2/3 列
   - 也可 Shift+点击“冻结”输入任意冻结列数
4. 拖拽选择一个矩形区域，点“复制选区”（粘贴到 Excel/记事本验证 TSV）
5. 在表格单元格/表头右键：复制单元格/整行/整列、隐藏列、冻结到此列
6. 勾选“复制带表头”后，再复制选区应包含表头行

### G3. 参数（绑定参数）
1. 打开【高级】→【参数】，新增参数 `p=1`
   - 可尝试把类型切到 text/int/real/null，验证类型控制有效
2. 执行：`SELECT :p AS v;` 或 `SELECT @p AS v;`
3. 打开【导出SQL结果】导出 CSV/XLSX，确保导出结果正确带参
4. 开启“统计总行数（COUNT(*)）”，确保 COUNT 查询不报错

**验收点：**
- 执行/分页/COUNT/导出均使用后端绑定参数（不应出现字符串替换导致的语法/引号问题）

### G4. 取消执行 + 强制恢复
1. 执行一个明显会跑很久的 SQL（或人为构造慢查询）
2. 点击【取消执行】，观察是否能尽快返回“已取消”
3. 若仍卡住，点击【强制恢复】，确认后应恢复可执行状态

**验收点：**
- 取消后 UI 状态恢复（按钮可用）
- 强制恢复后：实验事务关闭、连接重建、可继续执行其它 SQL

### G5. 脚本模式（多语句）
1. 开启【脚本模式】，执行：
   - `SELECT 1; SELECT 2;`
2. 观察状态栏脚本进度（1/2、2/2）

### G6. EXPLAIN / 索引 / 落地
1. 点击【EXPLAIN】查看计划 + SCAN/SEARCH 诊断
2. 打开【索引管理】：创建/删除索引，字段顺序输入生效
3. 点击【落地结果】：创建视图/临时表，返回后表/视图列表应刷新

## H. 项目中心 / 项目管理（重构后）冒烟清单

### H1. 启动默认主页
1. 启动程序后，默认应进入【项目中心】并落在“当前活动项目”视图
2. 若存在最近项目，应自动加载最近项目（但仍停留在项目中心）

### H2. 项目中心日常流
1. 在【项目中心-总项目概况】选择一个项目，点击“访问/加载”
2. 确认不会强行跳到【项目管理】页面（仍留在项目中心）
3. 在“当前活动项目”卡片里点击：
   - 编辑项目（源文件关系）
   - 导入/重导数据库
   - SQL 实验室

### H3. 源文件清单：保存/重导一致性
1. 在【项目管理-源文件清单】添加 Excel、勾选多个工作表、指定主表、修改显示名
2. 点击单个源文件按钮：
   - 未保存项目时应显示“保存数据库”，并能自动创建项目并导入
   - 已保存项目时应显示“重导数据库”
3. 关键验收点：点击后“源文件关系/主表/显示名”等配置会同步保存到项目（无需再手动点“保存项目”）

### H4. 项目管理：导入向导与高级区
1. 在【导入/重建】页点击“导入向导”，选择“重建/追加/合并”，应跳转并切换到对应 Tab
2. 点击“更多▼”菜单，能看到“重建项目库（危险）”
