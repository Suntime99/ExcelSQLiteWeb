> 说明：这是基于我们当前讨论的“项目中心/项目编辑/建模/数据库管理”等重构思路，把系统功能按**对象层级与职责边界**梳理成一张思维导图，便于后续做页面整合与按钮收敛。

```mermaid
mindmap
  root((ExcelSQLite 系统))
    核心对象/状态机
      活动上下文(Active Runtime)
        当前活动项目(Active Project)
        当前活动数据库连接(Active DB Connection)
        当前主表(Main Table)
        当前模式/权限(普通/专家)
      项目(Project)
        项目元信息(name/path/备注/时间)
        项目配置(Project Config)
          源文件关系(Source Bindings)
            文件列表
            工作表选择
            角色(main/dim/lookup...)
            目标表名/别名映射
          导入策略(Import Strategy)
            重建(全量重导)
            追加(Append)
            合并(Upsert/Merge)
              关键键(Key)
              冲突策略
          建模配置(Model Config)
            MTJ/多表关联草稿
            锁定SQL/视图
            字段/别名字典(可选)
      项目数据库(Project DB)
        SQLite 文件(持久化)
        临时库(会话内存库)
      数据库对象(DB Objects)
        表(Table)
        视图(View)
        索引(Index)
        触发器/其它(可选)

    页面/模块(重构后的职责分工)
      项目中心(默认主页)
        目标: 日常加载/切换/进入工作状态
        动作
          新建项目
          加载项目
          切换项目
          编辑活动项目(跳转项目编辑页)
          加载数据库(将 Project DB -> Active DB)
          快捷重导数据库(可选跳转导入向导)
        面板: 项目信息(Project Info)
          概况(源文件数量/主表/表数量/最后导入时间)
          状态(是否已加载DB/是否有未保存配置/是否有未提交事务)

      项目编辑(源文件关系编辑器)
        目标: 新增/编辑项目配置(可复现)
        分区建议
          基础信息(项目名默认/备注/路径只读或高级)
          源文件关系(文件-工作表-角色-表名/别名)
          导入向导入口(重建/追加/合并)
        关键规则
          保存项目=保存配置/关系
          导入/重导数据库=生成/更新 Project DB(产物)
          若无项目: 导入后应提示"保存为项目"或明确是临时库

      建模管理(语义层/分析层)
        目标: 基于已加载DB做关系/模型/视图
        子模块
          关联关系识别
          多表关联(MTJ/MTQ)
            生成草稿
            锁定SQL
            生成视图/模型(可选)
          统计/指标模型(可选扩展)
        产出
          写入项目配置(模型定义)
          可选写入DB对象(视图)

      数据库管理(DB 生命周期)
        目标: 管数据库本身(连接/重建/备份/导入目标)
        动作收敛(建议只保留三类)
          保存项目(配置)
          导入/重导数据库(产物)
          加载数据库(运行时)
        高级/更多
          备份/恢复(可选)
          清理/重建(危险操作)
          失败日志/诊断

      对象管理/索引管理(DB 运维)
        目标: 管 DB Objects(表/视图/索引)
        功能
          对象列表(表/视图)
          依赖分析
          重命名/删除
            自动更新项目配置引用(报告)
          索引管理
            创建/删除
            冗余提示(前缀覆盖)
            与索引建议联动

      专家工具(工程师工具箱)
        SQL 实验室/查询分析器
          执行/取消/强制恢复
          参数(后端绑定, 支持类型)
          EXPLAIN/索引建议/一键创建
          脚本模式(多结果集)
          结果表格(列宽/冻结/重排/右键/选区复制)
          导出SQL结果(Excel/CSV)
        说明
          面向调试/实验
          写入/DDL 需专家模式 + 风险提示 + 事务提交/回滚

      数据处理与分析(业务模块)
        元数据扫描
          指定文件夹/当前SQLite
          批量策略(每文件第一张/全部/关键字匹配)
        文件分析(Excel口径/SQLite口径)
        工作表分析
        数据清理(规则/发布包/回滚)
        差异校验(Excel vs SQLite/Cleansed)
        导出中心(报告/表/日志)

    典型用户路径(建议主流程)
      日常使用(90%)
        项目中心 -> 加载项目 -> 加载数据库 -> 进入分析/建模/查询模块
      配置变更
        项目中心 -> 编辑活动项目(源文件关系) -> 保存项目 -> 导入/重导数据库 -> 返回项目中心
      DB 运维
        项目中心 -> 数据库管理 -> 对象/索引管理
      工程调试
        专家工具 -> SQL 实验室(查询分析/索引/诊断/导出)
```

### 这张图用于解决的核心矛盾
1. **按钮/功能重复**：同一动作散落在“项目管理/数据库管理/源文件清单”等多个位置。
2. **职责混杂**：项目概况、项目编辑、数据库生命周期、对象运维、建模配置揉在一起。
3. **状态不透明**：用户不知道“改了配置是否已保存”“数据库是否已更新”“当前加载的是哪个库”。
