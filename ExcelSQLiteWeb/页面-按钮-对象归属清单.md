---
title: 页面-按钮-对象归属清单（用于重构收敛）
version: v0.1
---

目的：把现有 UI 的所有按钮/动作，强制归类到 **项目中心 / 项目信息 / 项目编辑 / 建模管理 / 数据库管理 / 对象&索引管理** 之一，并标注其影响的核心对象（Config/DB/Active/Objects/Model），用于后续“先调（调整布局与职责）”的重构落地。

> 使用方法：你们对照现有页面，把每个按钮填入表格；若一个按钮命中多类对象，按“硬性判定法”拆分为“跳转/向导入口 + 在目标页执行”。

---

## 1. 硬性判定法（放置规则）
1) 改项目文件/配置（Project Config）→ **项目编辑 / 建模管理**  
2) 生成/替换 SQLite 产物（Project DB）→ **数据库管理**  
3) 切换当前上下文（Active Project / Active DB）→ **项目中心**  
4) 改数据库对象（Tables/Views/Indexes）→ **对象管理/索引管理**

---

## 2. 核心对象字典（填表用）
- **Active**：当前活动项目、当前活动 DB 连接、当前主表、当前模式
- **Config**：项目配置（源文件关系、导入策略、建模配置、偏好设置…）
- **DB**：项目数据库文件（SQLite 文件作为产物）
- **Objects**：数据库对象（表/视图/索引/触发器…）
- **Model**：语义模型（MTJ、锁定 SQL、生成视图的定义等；通常写入 Config，可选生成 Objects）

---

## 3. 页面职责卡片（对照检查）
### 3.1 项目中心（默认主页）
- **只管**：Active（切换/加载/进入工作状态）
- **不管**：配置细节、导入细节、对象运维

### 3.2 项目信息（只读面板）
- **只管**：展示状态（Active/Config/DB/Model/Objects 的摘要）
- **不做**：执行会改变系统状态的动作（最多给跳转）

### 3.3 项目编辑（源文件关系）
- **只管**：Config（尤其 Source Bindings）
- **可提供**：导入向导入口（但执行导入属于 DB 管理）

### 3.4 建模管理
- **只管**：Model（写 Config 的模型子树；可选生成 View 对象）

### 3.5 数据库管理
- **只管**：DB 生命周期 + 加载到 Active
- 建议动作收敛为：**导入/重导**、**加载**、（高级）**备份/诊断**

### 3.6 对象管理/索引管理
- **只管**：Objects 运维（改名/删建/依赖/索引）
- 与 Config 引用更新联动（生成报告）

---

## 4. 动作清单（填表模板）

> 字段说明：  
> - **影响对象**：Active / Config / DB / Objects / Model（可多选，但多选意味着可能要拆动作）  
> - **目标页面**：重构后该动作最终应落在哪个页面执行  
> - **是否需要确认**：危险操作（删表/DDL/重建库/合并覆盖）必须确认  
> - **审计**：是否需要写入审计/报告（建议对所有写操作都记录）

| 现所在页面 | 现按钮/动作 | 影响对象 | 目标页面（重构后） | 是否需要确认 | 是否审计 | 备注（拆分/跳转/向导） |
|---|---|---|---|---|---|---|
| 项目中心 | 新建项目 | Active+Config | 项目中心→项目编辑 | 是（创建） | 是 | 创建后直接进入项目编辑 |
| 项目中心 | 加载项目 | Active | 项目中心 | 否 | 可选 | 切换 Active Project |
| 项目中心 | 加载数据库 | Active+DB | 项目中心（调用DB管理） | 否 | 可选 | 实际执行可复用 DB 管理逻辑 |
| 项目编辑 | 保存项目 | Config | 项目编辑 | 否 | 是 | 保存配置/关系 |
| 项目编辑 | 重导已选 | Config+DB | 项目编辑→数据库管理向导 | 是 | 是 | 必须：先保存源文件关系，再导入 |
| 建模管理 | 锁定SQL/生成MTJ | Model(+Config) | 建模管理 | 是（覆盖） | 是 | 写入模型定义 |
| 数据库管理 | 重导数据库 | DB | 数据库管理 | 是 | 是 | 生成/替换 Project DB |
| 对象管理 | 重命名表/视图 | Objects(+Config引用) | 对象管理 | 是 | 是 | 必须生成引用更新报告 |
| 索引管理 | 创建/删除索引 | Objects | 索引管理 | 是 | 是 | 与索引建议联动 |

> 你们把实际按钮逐行补齐后，我可以再基于表格给出“去重/合并/挪位”的具体调整建议清单。

## 5. 建议的“先调”（不大改架构）落地顺序
1) **先收敛按钮语义**：同义按钮合并为一个（其余改成跳转/更多菜单）
2) **先做状态透明**：在项目信息里明确显示：配置是否已保存、DB 是否已重导、是否已加载
3) **再做向导化**：把“重建/追加/合并”收进向导（项目编辑/数据库管理二选一落点）
4) **最后再做页面整合**：项目中心与项目信息的布局合并、项目编辑做成编辑器化
