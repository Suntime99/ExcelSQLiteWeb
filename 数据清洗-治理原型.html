模块拆分建议（先不定死）

你说“可能要拆”，这里给可选切法
方案 A：按“治理能力”拆（推荐）
Profiling / RuleSet / Run / Publish 为核心模块；“映射治理”是 P0 强模块（可独立页）。
3~4 个模块
方案 B：按“业务场景”拆
配置映射 / 单据记录 / 关联清理 三条线，每条线内部再有 Profiling/Rule/Run/Publish。
易重复建设
方案 C：按“对象”拆
字典表治理、映射表治理、主数据治理、交易明细治理…优点清晰，缺点是对象会膨胀。
需强边界
原型快捷键： 1~5切换左侧模块 C切换场景
